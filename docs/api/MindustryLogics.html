Building

An instance of a building.
This class contains all information about a building.
Do stuff with the building by accessing its member, for example,
b.Shoot(10, 10, true) will make building b shoot at (10, 10).

Example:
ExAmPle will be converted to @exAmPle.

Drawing

This class contains all drawing operations.

Coordinate inside displays:

  • The size of a Logic Display is 80x80.
  • (x = 0, y = 0) is the bottom left corner.
  • (x = 80, y = 80) is the upper right corner.
  • The size of a Large Logic Display is 176x176.
  • (x = 0, y = 0) is the bottom left corner.
  • (x = 176, y = 176) is the upper right corner.

Entity

An instance of an entity (building or unit).

ExcludeCreditAttribute

Filter

Create filters for methods that need filters.

Icon

This class contains the commonly used icon within Mindustry in string form.
For more icons use the string literal "\uXXXX" where XXXX is a hex number representing a Unicode character.
For example, "\uF800" is the icon of the unit Dagger.

InfoType

The information type of information of buildings and units.

But you can also get the information by accessing the members of a entity.

ItemType

KebabCaseType

Example:

LinkedBuilding

An instance of a building that is linked to the processor by name.
This class is created to solve the issue of needing to type GetLink(BuildingType, Int32)
everytime you wanted to get a linked building by its name.

Example declaration:
langword_csharp_private LinkedBuilding turret = GetLink(Duo, 27);
 Everytime the field turret is referenced, it will transpile to duo27.

RadarFilter

Filter for radar methods.
Use Radar() to create a filter than add constraints into it. (maximum three constraints)

To filter enemy ground unit: Filter.Radar().Enemy.Ground
To filter anything: Filter.Radar() or null

SortMethod

SortMethod for radar methods.

To sort by health: SortMethod.Health
